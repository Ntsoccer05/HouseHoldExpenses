# 複数選択コピー機能

## 🎯 概要

**背景**: 現状ではカレンダービュー内の支出を1件ずつコピーする必要があり、複数の支出を別の日付に繰り返す場合が手間。

**目標**: 同一日付内の複数支出をチェックボックスで選択して、1回の操作で別の日付へ一括コピーできるようにする。

---

## 📋 機能要件

#### PC版（デスクトップ）

1. **カレンダービューの日付セルをクリック**

2. **チェックボックス表示タイミング**
   - オプション1: 右パネルの「内訳」セクションにチェックボックスを常時表示
   - オプション2: 長押しまたはメニューで「複数選択モード」に切り替え
   
   **採用**: オプション1（常時表示で分かりやすい）

3. **複数選択後のUI**
   ```
   右パネルに表示:
   ┌─────────────────────────┐
   │ 2026-03-14 の支出        │
   │                          │
   │ ☑ 食費    ¥30,529       │
   │ ☐ 住居費  ¥100,080      │
   │                          │
   │ [選択状況: 1件選択]      │
   │ [コピー]  [キャンセル]   │
   └─────────────────────────┘
   ```

4. **コピー先日付の選択方法**
   - カレンダーピッカー（推奨）:
     ```
     「コピー先の日付を選択」
     ┌─────────────────┐
     │ 2026-04-14   📅 │  ← テキスト入力またはカレンダーポップアップ
     └─────────────────┘
     ```
   - またはテキスト入力（YYYY-MM-DD）
   - 両対応で使いやすさUP

#### SP版（スマートフォン）

1. **チェックボックス表示**
   - SP版の右パネルでもチェックボックスを表示

2. **コピー先選択**
   - モーダルダイアログで「コピー先の日付を選択」
   - カレンダーピッカーをモーダルで表示

### 2.2 機能要件

#### ユースケース1: 基本的なコピー

1. ユーザーが日付セルをクリック
2. 右パネルに支出一覧 + チェックボックスを表示
3. 複数支出を選択（☑チェック）
4. 「コピー」ボタンをクリック
5. 「コピー先の日付を選択」ダイアログを表示
6. 日付を選択 → 「コピー」ボタンをクリック
7. バックエンドでコピー処理実行
8. 完了メッセージ表示 → パネルを閉じる

**流れ（実装面）**:
```
Frontend:
  1. GET /api/contents?user_id=X&recorded_at=2026-03-14
     → 該当日付の全支出を取得
  
  2. ユーザーが複数選択
  
  3. POST /api/contents/copy
     {
       "source_date": "2026-03-14",
       "destination_date": "2026-04-14",
       "content_ids": [1, 3, 5]  ← 選択した支出のID
     }
     
  4. バックエンドでコピー実行
     → 新しいレコード作成（id は新規採番）
```

#### ユースケース2: キャンセル

- 「キャンセル」ボタン → チェックボックスをクリア → パネルを閉じる

#### 制約事項

1. **コピー可能な日付**
   - コピー先の日付は「過去・未来どちらでも可」
   - 同じ日付へのコピーはエラー（「異なる日付を選択してください」）

2. **コピー時の動作**
   - 新規レコード作成（既存データは変更しない）
   - `id` は新規採番
   - `created_at`, `updated_at` は現在時刻
   - `recorded_at` はコピー先の日付

3. **コピー対象**
   - contentテーブルの全カラム（content, amount, category_id, type_id など）
   - **コピー対象外**: id, created_at, updated_at

## 🗄️ データベース設計

### 既存テーブルの利用

**contentsテーブル**: 変更なし
```
id, user_id, type_id, category_id, amount, content, recorded_at, created_at, updated_at
```

**新規テーブル**: なし（複数選択コピーは既存テーブルで実現可）

## 🔌 API 設計

### エンドポイント1: 支出一覧取得（既存、条件追加）

```
GET /api/contents?user_id={user_id}&recorded_at={YYYY-MM-DD}

レスポンス:
{
  "data": [
    {
      "id": 1,
      "user_id": 1,
      "type_id": 2,  // 支出
      "category_id": 5,  // 食費
      "amount": 30529,
      "content": "スーパー",
      "recorded_at": "2026-03-14",
      "created_at": "2026-03-14T10:30:00Z",
      "updated_at": "2026-03-14T10:30:00Z"
    },
    ...
  ]
}
```

### エンドポイント2: 複数コピー実行（新規）

```
POST /api/contents/copy-multiple

リクエスト:
{
  "user_id": 1,
  "source_date": "2026-03-14",
  "destination_date": "2026-04-14",
  "content_ids": [1, 3, 5]  // コピー対象の支出ID
}

レスポンス:
{
  "success": true,
  "message": "3件の支出をコピーしました",
  "copied_count": 3,
  "created_ids": [101, 102, 103]  // 新規作成されたID
}

エラーレスポンス:
{
  "success": false,
  "message": "コピー先の日付は異なる日付を選択してください",
  "error_code": "SAME_DATE_ERROR"
}
```

## 🚀 実装方針

### フロントエンド（React）

1. **支出リスト表示コンポーネント修正**
   - チェックボックスの追加
   - 選択状況の表示更新

2. **日付ピッカーコンポーネント**
   - Material-UI の DatePicker を使用
   - またはカレンダーポップアップコンポーネント

3. **モーダルダイアログ**
   - Material-UI Dialog を使用

4. **API呼び出し**
   ```javascript
   const copyMultiple = async (sourceDate, destinationDate, contentIds) => {
     const response = await axios.post('/api/contents/copy-multiple', {
       source_date: sourceDate,
       destination_date: destinationDate,
       content_ids: contentIds
     });
     return response.data;
   };
   ```

### バックエンド（Laravel）

1. **コントローラーメソッド新規作成**
   - `ContentController@copyMultiple()`

2. **バリデーション**
   - `source_date != destination_date`
   - `content_ids` の全件が該当ユーザーに属するか確認

3. **トランザクション処理**
   ```php
   DB::transaction(function () {
     // content_ids をループ
     // 各recordで新規レコード作成
     // recorded_at だけコピー先日付に変更
   });
   ```

4. **API レスポンス**
   - 成功時: `success: true, copied_count: X, created_ids: [...]`
   - エラー時: 適切なエラーメッセージ
