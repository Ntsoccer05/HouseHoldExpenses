# 要求内容：複数選択コピー機能

## 概要

同一日付内の複数支出をチェックボックスで選択して、1回の操作で別の日付へ一括コピーできる機能を実装します。現状の「1件ずつのコピー」から「複数同時コピー」への改善により、ユーザーの手動操作を大幅に削減します。

## 背景

現状では、カレンダービュー内の支出を1件ずつコピーする必要があり、複数の支出を別の日付に繰り返す場合が手間になっています。特に、同じ日付で複数種類の支出を繰り返す場合（例：毎月1日に複数の固定費を登録）に、この手間が顕著です。

## 実装対象の機能

### 1. チェックボックス UI の追加

- 右パネルの「内訳」セクションにチェックボックスを常時表示
- 支出ごとにチェック・アンチェック可能
- 選択状況をリアルタイムに表示（「選択: X件」）

### 2. 複数選択後のコピー実行

- 「コピー」ボタンをクリック
- モーダルダイアログで「コピー先の日付を選択」
- 日付ピッカー（テキスト入力 + カレンダーピッカー両対応）で日付選択
- 「コピー」ボタンで実行

### 3. バックエンド複数コピー API

- `POST /api/copyMultipleContents` エンドポイント
- リクエスト: `source_date`, `destination_date`, `content_ids`
- レスポンス: `success`, `copied_count`, `created_ids`

### 4. エラーハンドリング

- 同じ日付へのコピーはエラー
- 存在しない content_id はエラー
- 権限外のコンテンツはエラー
- 無効な日付フォーマットはエラー

## 受け入れ条件

### チェックボックス UI

- [ ] 右パネルの支出一覧にチェックボックスが表示される
- [ ] チェック・アンチェックでリアルタイムに「選択: X件」が更新される
- [ ] PC版・SP版両方で動作する
- [ ] チェックボックスの状態は、データベースに保存されない（一時的）

### 複数コピー実行

- [ ] 「コピー」ボタンをクリックするとモーダルが表示される
- [ ] 日付ピッカーで日付を選択できる
- [ ] 「コピー」ボタンで複数支出が一括コピーされる
- [ ] コピー完了後、成功メッセージが表示される
- [ ] コピー後、カレンダーが更新されて新しい支出が表示される

### API エンドポイント

- [ ] `POST /api/copyMultipleContents` エンドポイントが動作する
- [ ] 正常なリクエストに対して正しいレスポンスが返される
- [ ] 同じ日付へのコピーでエラーが返される
- [ ] 権限外のコンテンツに対してエラーが返される

### 既存機能への影響

- [ ] 既存の1件コピー機能は引き続き動作する
- [ ] 既存の支出表示・編集・削除は影響を受けない
- [ ] 既存のカテゴリ機能は変更されない

## 成功指標

### 定量的目標

- ユーザーが 5 件の支出をコピーする際の操作ステップが 50% 削減される
  - 現状: 支出1件 × 5回 + コピー先日付入力 × 5回 = 10 ステップ
  - 目標: チェック × 5回 + 日付入力 × 1回 = 6 ステップ

### 定性的目標

- ユーザーが複数支出の一括コピーを「簡単」と感じられる
- エラーメッセージが明確で、ユーザーが対応を理解できる

## スコープ外

以下はこのフェーズでは実装しません：

- 複数コピーの予約（定期実行）
- コピー履歴の記録
- 「このシリーズすべてをコピー」機能
- ドラッグ&ドロップによるコピー
- カレンダー上での複数選択
